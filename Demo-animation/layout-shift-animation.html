Explanation

This demo shows how animations can cause layout shifts and affect performance.

In this example, the .box element is animated to move from the top-left corner to the bottom-right corner of the viewport. This movement causes a layout shift.

The animation triggers layout and paint steps in the browser's rendering pipeline, which are more expensive than animating properties like transform and opacity that only affect compositing.

Animating properties that affect layout or paint can lead to performance issues and a less smooth experience, as the browser has to recalculate the geometry and repaint the affected elements.

For optimal performance, it's recommended to animate properties that only affect compositing, such as transform and opacity, to achieve smoother animations and avoid layout shifts.
