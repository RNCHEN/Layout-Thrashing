Explanation

This demo shows how to fix layout shifts caused by animations by using properties that only affect compositing.

In this example, the .promotion-box element is animated using the transform property, which only affects compositing and does not trigger layout or paint steps in the browser's rendering pipeline.

The @keyframes moveBox animation defines the movement of the box using transform: translate(300px, 600px);, which ensures smooth animations without causing layout shifts.

Animating properties like transform and opacity helps achieve smoother animations and better performance, as the browser can handle these changes more efficiently.
